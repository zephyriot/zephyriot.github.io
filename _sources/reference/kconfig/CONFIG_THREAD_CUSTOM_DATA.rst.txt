.. option:: CONFIG_THREAD_CUSTOM_DATA:
.. _CONFIG_THREAD_CUSTOM_DATA:

This option allows each thread to store 32 bits of custom data,
which can be accessed using the k_thread_custom_data_xxx() APIs.
